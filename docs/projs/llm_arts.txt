首先, post-norm 已经基本不再使用, pre-norm 已经基本是标准做法: pre-norm 给 input x 留出了"resnet通路".
在预训练中, pre-norm 的 loss/bleu 等都显著优于 post-norm, 梯度衰减更弱, 梯度传播更稳, 梯度的 global norm spikes 整体更少
pre-norm 最开始是为了去掉warmup, 但现在更重要的作用是使得训练更稳定、能使用更大的学习率在大网络上.
(warmup: 以前用post-norm时, 由于原始数据+随机初始化的权重, norm又放在最后, 导致中间激活值可能忽大忽小, 数值波动非常激烈, 导致梯度值极大. 若配合大学习率, 提高了NaN出现的概率.
所以warmup过程就是在训练的早期阶段(batch或epoch)用“逐步增大至目标值的小学习率”来训, 避免过程训崩。pre-norm可以很好地规整数据)

其次, RMSNorm 在崛起: LayerNorm对特征维度内的分量, 作mean-std标准化后, 作参数scale+bias; 而RMSNorm的计算要简单很多, 它对特征维度内的分量, 作原点标准化后, 仅作参数scale
RMSNorm相对LayerNorm, 其计算量更少(没有求均值), 参数量也更少(没有bias), 效果却不差.
问题: normalization操作在整体 FLOPs&Memory 统计中，占比极小, 矩阵运算才是大头, RMSNorm相对LayerNorm这点节省有意义吗？
答案: 有意义, 因为FLOPs不是runtime, 具体看整体 runtime 统计, normalization操作的占比是相当可观的，所以 RMSNorm 在 runtime 上的节省是显著的. 而且RMSNorm效果不差(经过严格验证).

第三, FFN 中的 bias 参数普遍被舍弃. 原因也是出于 memory 和 优化时的稳定性.



激活函数的 Gated化:
激活函数(Linear Unit)比如 ReLU、GeLu 等，都是指给线性变换 x@W+b 加上 element-wise 的非线性函数. 那么 Gated Linear Unit 就是在 非线性之后,
再哈德玛德乘积 一个x的线性变换 x@V+a, 然后再将结果送入下一层. 即:
原有 GeLU of FF(x) := gelu(x @ W_1 + b_1) @ W_2 + b_2
gated化后 GeGLU of FF(x) := ( gelu(x @ W_1 + b_1) hadmad (x @ V + a) ) @ W_2 + b_2

总结:
普通FFN: x(d_mod 维度) --线性映射--> d_ff 维度(通常是4*d_mod) --非线性化-->--线性映射--> d_mod 维度
门控FFN: x(d_mod 维度) --线性映射--> 2*d_ff 维度(通常是8*d_mod) --双路交积(一半d_ff经非线性化后哈德玛德乘另一半d_ff)-->--线性映射--> d_mod 维度
这里典型的非线性化有 GeLU/ReLU/Swish等. Swish(x) = x * sigmoid(x)

引入门控在训练上是有效果的, 尽管它不是必需品.
如果模型不在隐藏维度作 scale down, 那么对比 vanilla 和 GLU variants 的 FLOPs(以per-row计)
vanilla: 2*d_model*d_ff
GLU variant: 2*d_model*d_ff + d_model*d_ff = 3*d_model*d_ff
若GLU variant保持 FLOPs 不变, 那么其 d_ff 应该是 vanilla版本 的 2/3 scale down
如果取经典的 d_ff = 4*d_model, 那么 GLU variant 的 d_ff 应该等于 8/3*d_model = 2.67*d_model




MLP 和 attention 的并行化:
当前 transformer block 是序列执行的, 先 LayerNorm+Attention, addLayerNorm之后再MLP: y = x + MLP( LayerNorm(x + Attention(LayerNorm(x))) )
有些模型并行化 MLP 和 attention: y = x + MLP(LayerNorm(x)) + Attention(LayerNorm(x))
并行MLP+attention的计算速度更高. 消融实验展示效果有一定折扣



位置编码: absolute-position-encoding --> rotary-position-encoding
RoPE 是 apply 在 query 和 key 上的: 对奇/偶维度作两两二维旋转 --> new query 和 key



超参数 hyperparameters
1. FFN 里的 增维维度 应该是 模型维度 的多少? 答: 几乎所有模型都把 FFN 里的 d_ff 设为 4倍 d_model. 此系数几成共识, 除非
    GLU门控: 上面论述过, 为了保持 FLOPs, GLU variant 应该作 2/3 scale down. 如此 d_ff 约为 2.67倍 d_model. 实际上流行模型的系数在 2.5 - 4 之间
    T5(原始版本): T5的 d_ff = 65536 而 d_model = 1024, 比例达到64 --> T5 的FFN是一个超宽的神经网络 <--- scale up d_ff 的原因是谷歌的TPU非常适应超大稠密矩阵乘法
   从经验上看, d_ff / d_model 在 1至10 这个区间内, 训练效果比较好. 原始T5的 64 过大了, T5 v1.1 马上使用了一个正常地多的值 2.5
   ---> 总结：default choice d_ff / d_model = 4 for 普通模型, d_ff / d_model = 2.67 for GLU variant
